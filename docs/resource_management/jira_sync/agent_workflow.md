# Agent Workflow

전체 구성원이 직접 Markdown 본문을 수정하게 하기보다, 입력은 간단히 하고 반영은 에이전트가 수행하게 만드는 편이 낫다.

## 추천 역할

### 1. 구성원 입력 에이전트

- 구성원이 "내 일정 추가", "기간 변경", "배정률 변경" 요청
- 입력 형식을 표준화
- 누락 값 확인

입력 예시:

```text
이름: 홍길동
업무명: 주문/결제 안정화
유형: ops
JIRA: TASK-451
시작: 2026-05-02
종료: 2026-05-20
배정률: 20%
```

### 2. 일정 반영 에이전트

- 월간 파일 업데이트
- 개인 파일 업데이트
- 충돌 여부 확인

### 3. 요약판 생성 에이전트

- 월간 파일을 기준으로 팀별/임원용 요약판 갱신
- Mermaid Gantt 재생성

## 권장 처리 흐름

1. 구성원이 일정 등록/수정 요청
2. 입력 에이전트가 표준 포맷으로 변환
3. 일정 반영 에이전트가 월간 파일과 개인 파일 갱신
4. 충돌 검사:
   - 개인 총 배정률 100% 초과
   - 동일 기간 중복 핵심 역할
   - 시작일/종료일 누락
5. 요약판 생성 에이전트가 상위 문서 갱신
6. 팀 리더가 최종 확인

## 스킬 설계 포인트

- 자연어 입력을 표준 데이터로 변환
- 월간 파일에서 해당 월만 수정
- 분기 파일은 월간 파일 변화가 누적될 때만 갱신
- JIRA와 충돌하면 JIRA를 원장으로 간주

## 현실적인 운영 권장안

- 구성원: 자연어 요청
- 에이전트: Markdown 초안 생성
- 팀장/리드: 승인
- 승인 후: 월간/분기/대시보드 반영

완전 자동 덮어쓰기보다 "자동 초안 + 사람 승인" 방식이 운영 안정성이 높다.
