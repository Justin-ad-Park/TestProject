# 썸네일 서브에이전트 만들기

## 1. 문서 목적

이 문서는 **쇼핑몰용 썸네일 이미지를 생성하는 간단한 서브에이전트**를  
Codex의 공식적인 **Subagents + Skills** 방식에 맞춰 어떻게 설계하면 좋은지 정리한 초안입니다.

이번 초안의 목표는 아래 4가지를 동시에 만족하는 것입니다.

1. 디자이너가 이해할 수 있는 운영 문서일 것
2. Codex 공식 구조에 맞는 서브에이전트 초안일 것
3. 스킬과의 역할 분리가 분명할 것
4. 이후 크기, 여백, 로고 규칙을 쉽게 바꿀 수 있을 것

---

## 2. 이번에 만들 서브에이전트의 요구사항

### 기본 기능

1. 사용자가 이미지를 첨부하고 썸네일 생성을 요청하면 `600px x 600px` 결과물을 만든다.
2. 결과 이미지의 좌측 상단에 로고를 넣는다.
3. 로고 배치 규칙은 아래와 같다.

- 좌측 여백: `25px`
- 상단 여백: `25px`
- 로고 폭: `200px`
- 로고 비율: 원본 비율 유지
- 사용 로고: [pulmuone-shop.svg](https://shop.pulmuone.co.kr/assets/pc/images/logo/pulmuone-shop.svg)

### 변경 가능해야 하는 값

아래 값은 하드코딩하지 않고, 서브에이전트 구성 정보 또는 연결된 스킬 구성 정보에서 바꿀 수 있어야 합니다.

- 썸네일 가로 크기
- 썸네일 세로 크기
- 로고 좌측 여백
- 로고 상단 여백
- 로고 폭
- 저장 루트 경로
- 날짜별 하위 폴더 규칙
- 파일명 규칙

## 3. Codex 공식 구조 기준으로 어떻게 나눌까

이번 프로젝트에서는 아래 3개를 구분해서 이해하는 것이 좋습니다.

### 1) `CODEX.md`

이 프로젝트에서 Codex가 따라야 하는 **프로젝트 공통 안내 문서**입니다.  
공식 문서의 `AGENTS.md`와 비슷한 역할로 이해하면 됩니다.

이 문서에는 아래를 넣는 것이 좋습니다.

- 프로젝트 목적
- 어떤 서브에이전트가 있는지
- 어떤 스킬을 우선 사용하는지
- 결과물 저장 위치
- 디자인팀 공통 원칙

즉, `CODEX.md`는 **프로젝트의 진입점이자 연결 허브**입니다.

### 2) `Subagent`

공식 문서 기준으로 서브에이전트는 `.codex/agents/*.toml` 파일로 정의합니다.  
필수 필드는 아래 3개입니다.

- `name`
- `description`
- `developer_instructions`

필요하면 아래도 함께 정의할 수 있습니다.

- `model`
- `model_reasoning_effort`
- `sandbox_mode`
- `skills.config`

서브에이전트는 "누가 이 일을 맡는가"를 정의합니다.

### 3) `Skill`

공식 문서 기준으로 스킬은 `SKILL.md`를 중심으로 하는 **재사용 가능한 워크플로우 형식**입니다.  
필요하면 `assets/`, `references/`, `scripts/`를 함께 둘 수 있습니다.

스킬은 "이 일을 어떤 절차와 규칙으로 수행하는가"를 정의합니다.

---

## 4. 이번 프로젝트의 권장 구성 방식

이번 썸네일 작업은 아래처럼 **`CODEX.md` + Subagent + Skill** 3층 구조로 시작하는 것을 권장합니다.

### 권장 구조

1. `CODEX.md`
2. `.codex/agents/thumbnail_maker.toml`
3. `.agents/skills/thumbnail-maker/SKILL.md`

### 역할 분리

`CODEX.md`

- 프로젝트 전체 문맥 제공
- 어떤 서브에이전트가 있는지 안내
- 스킬 사용 원칙 연결

`subagent`

- 썸네일 작업 전용 담당자 정의
- 언제 써야 하는지 정의
- 어떤 규칙을 반드시 지켜야 하는지 정의

`skill`

- 썸네일 제작 절차 정의
- 크기/로고/여백 구성값 정의
- 저장 위치/파일명 규칙 정의
- 실패 처리 방식 정의

즉,

- `CODEX.md` = 프로젝트 운영 안내
- `subagent` = 역할
- `skill` = 작업 방법

으로 분리하면 이후 유지보수가 가장 쉽습니다.

```text
/CODEX.md

/.agents
  /skills
    /thumbnail-maker
      SKILL.md
      /assets
        pulmuone-shop.svg
      /config
        thumbnail.toml
      /references
        thumbnail-spec.md
```

### 중요한 포인트

로고를 URL로만 참조하기보다,  
초기 버전에서는 `assets/pulmuone-shop.svg`로 **프로젝트 안에 함께 보관**하는 것을 권장합니다.

이유:

- 런타임에서 외부 URL 접근이 항상 보장되지 않을 수 있음
- URL 변경 또는 접근 실패 리스크가 있음
- 로고 버전 관리가 쉬워짐

## 6. 썸네일 서브에이전트 초안 설계

### 역할 정의

이 서브에이전트는 **첨부된 원본 이미지를 바탕으로 쇼핑몰용 정사각형 썸네일 이미지를 생성**하는 역할만 담당합니다.

### 서브에이전트가 해야 하는 일

- 입력 이미지를 기준으로 정사각형 썸네일을 만든다
- 구성 정보에 정의된 크기로 결과물을 만든다
- 좌측 상단 지정 위치에 로고를 삽입한다
- 로고 비율은 유지하고 지정 폭에 맞춰 배치한다
- 구성 정보에 정의된 경로와 파일명 규칙으로 결과물을 저장한다

### 서브에이전트가 하지 말아야 하는 일

- 임의로 로고 위치를 바꾸지 않는다
- 임의로 로고 비율을 찌그러뜨리지 않는다
- 지정된 출력 크기를 무시하지 않는다
- 과도한 배경 연출이나 장식 추가를 기본 동작으로 삼지 않는다

## 7. Subagent 초안

아래는 Codex 공식 subagent 형식에 맞춘 초안입니다.

```toml
name = "thumbnail_maker"
description = "첨부된 이미지로 쇼핑몰용 정사각형 썸네일을 만드는 서브에이전트."
model = "gpt-5.4"
model_reasoning_effort = "medium"
developer_instructions = """
You are a shopping-mall thumbnail subagent.

Your job is to create a square thumbnail from an attached source image.

Always follow the repository thumbnail skill when it is available.

Rules:
- Respect configured output width and output height.
- Respect configured logo left offset, top offset, and logo width.
- Respect configured output root directory, date-based subdirectory rule, and filename rule.
- Preserve the logo aspect ratio.
- Do not invent alternative logo placements unless the user explicitly asks for a different layout.
- If required inputs are missing, report exactly what is missing.
- If the requested output cannot be produced exactly, explain the limitation and propose the closest safe output.
"""

[[skills.config]]
path = ".agents/skills/thumbnail-maker/SKILL.md"
enabled = true
```

### 메모

위 예시는 초안이며, 실제 모델명은 운영 비용과 속도 기준으로 조정할 수 있습니다.

## 8. Skill 초안

스킬은 서브에이전트의 실제 작업 절차를 담습니다.

```md
---
name: thumbnail-maker
description: 첨부 이미지를 바탕으로 로고가 포함된 쇼핑몰용 정사각형 썸네일을 생성해야 할 때 사용하는 스킬.
---

# Thumbnail Maker

## Purpose

Create a square shopping thumbnail from an attached source image.

## Configuration

- output_width: 600
- output_height: 600
- logo_asset_path: .agents/skills/thumbnail-maker/assets/pulmuone-shop.svg
- logo_left: 25
- logo_top: 25
- logo_width: 200
- output_root_dir: outputs/thumbnails
- output_subdir_pattern: YYYYMM/DD
- filename_rule: timestamp
- filename_timestamp_pattern: thumbnail_YYYYMMDD_HHMMSS.png
- filename_source_pattern: {source_name}.png

## Workflow

1. Confirm that the user attached at least one source image.
2. Read the thumbnail configuration values.
3. Create a square thumbnail using the configured output width and output height.
4. Place the configured logo at the top-left corner using logo_left and logo_top.
5. Scale the logo to logo_width while preserving the original aspect ratio.
6. Save the final thumbnail into the configured output directory.
7. Return the saved file path and final thumbnail output.

## Rules

- Do not distort the logo.
- Do not change configured spacing values unless the user asks for a configuration change.
- Prefer a clean composition that preserves the source subject clearly.
- If square conversion requires cropping, avoid cutting the main product unless explicitly allowed.
- Create the output directory if it does not exist.
- If filename rule is `timestamp`, save as `thumbnail_YYYYMMDD_HHMMSS.png`.
- If filename rule is `source_name`, save based on the original source filename.

## Missing Information Handling

- If no source image is attached, ask for the image.
- If logo asset is unavailable, report the missing asset path clearly.
- If the output format is unspecified, use PNG by default unless the project later defines another default.
- If source filename cannot be derived, fall back to the timestamp naming rule.
```

## 10. 변경 가능한 구성값은 어떻게 관리할까

이번 요구사항의 핵심은 **크기와 여백을 바꿀 수 있어야 한다**는 점입니다.  
따라서 이 값들은 설명 문장 안에 흩어두지 말고, **한 곳에 모아서 관리**해야 합니다.

### 추천안 A: `SKILL.md` 안의 Configuration 블록

가장 간단한 시작 방식입니다.

장점:

- 빨리 시작할 수 있음
- 디자이너가 읽기 쉬움
- 문서 기반 운영에 적합함

단점:

- 값만 따로 교체하기엔 구조화가 약함

### 추천안 B: 별도 설정 파일 분리

예:

```text
/.agents/skills/thumbnail-maker/config/thumbnail.toml
```

예시:

```toml
output_width = 600
output_height = 600
logo_left = 25
logo_top = 25
logo_width = 200
logo_asset_path = ".agents/skills/thumbnail-maker/assets/pulmuone-shop.svg"
default_output_format = "png"
output_root_dir = "outputs/thumbnails"
output_subdir_pattern = "YYYYMM/DD"
filename_rule = "timestamp"
filename_timestamp_pattern = "thumbnail_YYYYMMDD_HHMMSS.png"
filename_source_pattern = "{source_name}.png"
```

장점:

- 값 변경이 쉬움
- 이후 서브에이전트가 늘어나도 일관되게 관리 가능
- 채널별 썸네일 규격 확장에 유리함

단점:

- 초기 문서 수가 조금 늘어남

### 이번 단계의 권장 결론

처음에는 **문서 이해를 위해 `SKILL.md`에 설정을 두고**,  
실사용이 시작되면 빠르게 **별도 설정 파일로 분리**하는 것을 권장합니다.

## 12. 구현 시 주의할 점

### 1) `600x600`은 "최종 결과물 크기"인지 먼저 고정해야 함

OpenAI의 이미지 생성 모델은 시점에 따라 지원 해상도나 권장 해상도가 달라질 수 있습니다.  
따라서 실제 구현에서는 다음 중 하나를 선택해야 합니다.

- 생성 단계부터 정사각형 규격으로 만들고 최종 결과를 600x600으로 맞춘다
- 더 큰 정사각형으로 생성한 뒤 후처리로 600x600으로 리사이즈한다

초기 운영에서는 **최종 납품 규격을 600x600으로 고정**하고,  
생성 내부 해상도는 추후 구현 방식에 맞춰 분리하는 것이 좋습니다.

### 2) 로고 URL 직접 사용보다 로컬 자산화가 안정적임

현재 요구사항은 아래 URL의 로고를 사용합니다.

- [https://shop.pulmuone.co.kr/assets/pc/images/logo/pulmuone-shop.svg](https://shop.pulmuone.co.kr/assets/pc/images/logo/pulmuone-shop.svg)

하지만 실제 운영에서는 다음 중 하나가 필요합니다.

- 로고를 프로젝트에 저장
- 또는 런타임에서 외부 접근이 가능해야 함

초기 버전은 **프로젝트 내 로고 파일 저장**을 권장합니다.

### 3) 정사각형 변환 기준을 정해야 함

입력 이미지가 세로형 또는 가로형일 수 있으므로 아래 기준이 필요합니다.

- 중앙 크롭
- 배경 확장
- 여백 추가
- 피사체 우선 보호

이 기준이 없으면 같은 요청에도 결과가 흔들릴 수 있습니다.

### 4) 출력 포맷도 정하는 것이 좋음

현재 요구사항에는 파일 형식이 없습니다.  
초기값은 `PNG`를 권장하지만, 운영 목적에 따라 아래 중 하나로 정해야 합니다.

- PNG
- JPG
- WebP

## 14. 추가로 필요한 정보

아래 정보가 정해지면 실제 서브에이전트 품질이 훨씬 안정됩니다.

### 필수에 가까운 정보

1. 입력 이미지가 정사각형이 아닐 때 어떤 방식으로 맞출지

- 중앙 크롭
- 여백 추가
- 배경 확장
- 상품이 잘리지 않도록 자동 보호

2. 최종 파일 형식

- PNG
- JPG
- WebP

3. 결과 저장 방식

- 파일로 저장할지
- 채팅 결과로만 반환할지
- 저장 폴더와 파일명 규칙이 필요한지

4. 로고 파일을 프로젝트 내부에 저장할지

- 로컬 자산 사용
- URL 직접 참조

### 있으면 좋은 정보

1. 배경 처리 원칙

- 원본 배경 유지
- 배경 제거
- 단색 배경 적용

2. 썸네일 대상 종류

- 상품 단일컷
- 음식 연출컷
- 배너형 소재

3. 채널별 규격 확장 계획

앞으로 600x600 외에도 아래가 필요할 수 있습니다.

- 오픈마켓 썸네일
- 앱 리스트 썸네일
- 프로모션 카드형 썸네일

## 15. 권장 결론

이번 썸네일 작업은 아래 구조로 시작하는 것이 가장 현실적입니다.

1. `CODEX.md`에 프로젝트 맥락과 서브에이전트 연결 규칙을 둔다
2. `.codex/agents/thumbnail_maker.toml`로 썸네일 전용 서브에이전트를 만든다
3. `.agents/skills/thumbnail-maker/SKILL.md`로 작업 절차와 구성값을 관리한다
4. 크기와 여백값은 스킬의 구성 영역 또는 별도 설정 파일로 분리한다

이 방식이면 지금은 단순하게 시작할 수 있고,  
나중에는 채널별 썸네일 규격이나 브랜드별 로고 규칙까지 확장하기 쉽습니다.
