# Thumbnail Maker 한글 설명

## 목적

이 문서는 `.agents/skills/thumbnail-maker/SKILL.md`의 한글 설명본입니다.

사람이 스킬 내용을 쉽게 이해하기 위한 문서이며,  
에이전트 실행 기준은 항상 원본 스킬 문서인 `SKILL.md`를 따릅니다.

---

## 이 스킬이 하는 일

`thumbnail-maker`는 첨부된 원본 이미지를 바탕으로 쇼핑몰용 정사각형 썸네일을 생성하는 스킬입니다.

기본적으로 아래 작업을 수행합니다.

- 원본 이미지를 정사각형으로 중심 크롭
- 결과 크기를 설정값에 맞게 조정
- 좌상단에 풀무원 로고 오버레이
- 필요하면 하단에 라벨 오버레이
- 정해진 저장 경로와 파일명 규칙으로 저장

---

## 기준 설정 파일

이 스킬은 아래 설정 파일을 기준으로 동작합니다.

- `.agents/skills/thumbnail-maker/config/thumbnail.toml`

이 파일에는 아래 값들이 들어 있습니다.

- 출력 가로/세로 크기
- 로고 경로
- 로고 위치와 폭
- 저장 경로
- 파일명 규칙
- 라벨 위치와 간격
- 라벨 높이 모드
- 라벨 자산 경로

---

## 기본 작업 흐름

1. 사용자가 원본 이미지를 첨부했는지 확인
2. 설정 파일 로드
3. 로고 자산 존재 여부 확인
4. 요청에 `free_shipping`, `mix_and_match_discount` 또는 사용자 정의 라벨 텍스트가 포함됐는지 확인
5. 내장 키워드는 표시 텍스트로 변환하고, 사용자 정의 텍스트는 그대로 라벨 문구로 사용
6. 잠금된 썸네일 생성 규칙에 따라 이미지 생성
7. 저장 폴더 계산
8. 파일명 계산
9. 결과 저장
10. 저장된 경로 반환

---

## 잠금된 생성 규칙

이 스킬은 프로젝트 소유자가 명시적으로 변경하기 전까지 아래 방식을 유지해야 합니다.

### 구현 경로

- 표준 썸네일 생성 스크립트는 `scripts/create_thumbnail.swift`
- 로고는 설정 파일에 정의된 자산 경로를 기준으로 사용
- 라벨은 최종 합성 시점에 스크립트가 직접 그린다

### 이미지 처리 규칙

1. 원본 이미지 로드
2. 중심 크롭으로 정사각형 변환
3. 출력 크기(예: 600x600)로 리사이즈
4. 좌상단에 로고 배치
5. 로고 높이는 폭과 원본 비율로 계산
6. 라벨 요청이 없으면 라벨을 넣지 않음
7. 라벨 1개 요청 시:
   - 좌측 시작점: `label_left`
   - 하단 기준: 라벨의 보이는 하단선이 썸네일 하단에서 `label_bottom` 위
   - 라벨 높이: `label_height_mode = match_logo_height`이면 실제 로고 표시 높이와 동일
8. 라벨 2개 요청 시:
   - 첫 번째 라벨을 먼저 배치
   - 두 번째 라벨은 같은 바닥선에 배치
   - 두 번째 라벨 시작점은 `free_shipping.right + label_gap`
9. 라벨 폭은 텍스트 렌더 폭과 좌우 패딩으로 계산
10. 라벨은 최종 캔버스에 직접 그리며, 박스 밖 배경은 투명으로 유지
11. 기본 출력 포맷은 PNG

---

## 라벨 배치 규칙 설명

### 라벨이 없는 경우

- 아무 라벨도 추가하지 않음

### 라벨이 1개인 경우

- 썸네일 전체 캔버스의 좌하단을 기준으로 계산
- X 좌표는 좌측 경계에서 `label_left`만큼 떨어진 위치
- 라벨의 보이는 하단선이 하단 경계에서 `label_bottom`만큼 올라간 위치

즉, 코드상의 Y값은 내부 좌표계 기준으로 라벨의 시작 Y가 아니라,  
결과적으로 **라벨의 보이는 하단선이 썸네일 하단에서 `label_bottom` 위**에 오도록 계산해야 합니다.

### 라벨이 2개인 경우

- `free_shipping`을 먼저 좌하단 기준으로 배치
- `mix_and_match_discount`는 같은 Y축에 배치
- X는 `free_shipping`의 오른쪽 끝에서 `label_gap`만큼 띄워서 시작

### 사용자 정의 텍스트 라벨

- `free_shipping`은 `무료배송`으로 렌더링
- `mix_and_match_discount`는 `골라담아할인`으로 렌더링
- 그 외 텍스트는 사용자가 요청한 문구를 그대로 라벨 박스 안에 렌더링

---

## 기본적으로 바꾸면 안 되는 것

- 중심 크롭을 다른 방식으로 임의 변경
- 로고 위치를 임의 변경
- 로고 폭 규칙을 임의 변경
- 요청이 없는데 라벨을 자동 추가
- 라벨 1개를 가운데 정렬
- 두 라벨 순서를 변경
- 다른 구현 스크립트로 임의 교체
- 저장 경로 규칙 임의 변경

---

## 예외 처리 규칙

- 원본 이미지가 없으면 사용자에게 요청
- 로고 자산이 없으면 경로를 명확히 보고
- 지원되지 않는 라벨 키워드인지, 사용자 정의 라벨 텍스트인지 구분해서 처리
- 설정값과 요청이 충돌하면 설정 변경인지 이번 실행만 예외 처리할지 확인
