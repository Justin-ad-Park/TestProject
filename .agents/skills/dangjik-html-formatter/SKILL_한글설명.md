# 당직정리용 에이전트 한글 설명

이 문서는 사람이 `SKILL.md`의 내용을 빠르게 파악하기 위한 참고 문서입니다.

에이전트 작업 시 실제 기준 문서는 반드시 `SKILL.md`만 사용합니다.
이 문서는 에이전트 동작 기준이나 소스 오브 트루스가 아닙니다.

## 목적

당직 보고 메일 본문이나 이미지에서 사업장별 당직 내용을 추출해서,
기존 HTML 당직표의 해당 행에 맞는 형식으로 반영하는 스킬입니다.

핵심은 아래 두 가지입니다.

- 출력 형식을 항상 HTML로 유지
- 사업장마다 다른 표현을 공통 양식으로 통일

## 출력 기준

- 결과물은 HTML 유지
- 이메일 공유용 고정폭 표 형식 유지
- 셀 내부 줄바꿈은 반드시 `<br>` 사용
- 전체 표 구조나 디자인은 사용자가 요청하지 않으면 변경하지 않음
- 메일 발송용이면 템플릿 또는 사용자 요청에 맞춰 표 상단/하단에 메일 본문 문구를 함께 포함
- 메일 본문 문구는 현재 템플릿 문구를 그대로 따르고, `보고 드립니다` 같은 띄어쓰기까지 유지

## 입력값에서 뽑아야 하는 정보

각 보고서나 이미지에서 아래 정보만 뽑아서 한 행에 반영합니다.

- 사업장명
- 당직자
- 근무인원
- 생산 내용
- 외부방문
- 특이사항

이미지 보고서를 받았을 때는 이미지 표를 그대로 복사하지 않고,
필요한 사실만 추출해서 기존 HTML 행 형식으로 다시 정리합니다.

## 행 업데이트 원칙

1. 해당 사업장 행만 수정
2. 표의 행 순서는 유지
3. 상태 셀 내용은 해당 사업장 기준으로 전체 교체
4. 보고가 도착해 업데이트 완료된 행은 회색 해제
5. 아직 미업데이트인 행은 회색 유지

## 오전/오후 반영 규칙

- 오전 보고와 오후 보고가 모두 있으면 오전 보고로 초안을 만듭니다.
- 오후 보고는 오전 초안을 완전히 다시 쓰는 것이 아니라, 달라진 내용만 반영하는 업데이트로 취급합니다.
- 같은 사업장에서 재보고가 온 경우:
- 재보고 인원이 기존 초안보다 줄어들면 `근무인원`은 낮추지 않습니다.
- 재보고 인원이 더 크거나, 처음 확인된 신뢰 가능한 숫자라면 `근무인원`을 업데이트합니다.
- `생산`에 새로 확인된 내용이 있으면 오전 내용을 지우지 말고 기존 내용에 추가합니다. 단, 원문이 이전 내용을 명확히 정정하는 경우만 예외적으로 교체할 수 있습니다.
- `외부방문`에 새로 확인된 내용이 있으면 추가 또는 최신화합니다.
- `특이사항`은 재보고에서 실제 변경이 확인된 경우만 바꿉니다.
- 오후 보고가 오전 내용을 명확히 대체한다고 판단되지 않으면, 오전에 이미 확인된 사실은 유지합니다.

## 표현 통일 규칙

표 안의 표현은 아래처럼 통일합니다.

- `1. 근무인원`
- `2. 생산`
- `3. 외부방문`
- `4. 특이사항`

세부 치환 규칙:

- `외부 방문자`, `업체방문`, `외부방문 및 공사 업체` → `외부방문`
- `생산라인`, `생산현황` → `생산`
- 물류 작업 성격일 때만 `생산(물류작업)` 사용
- `안전, 환경`이 별도 운영 항목이 아니라 이상 유무를 뜻하는 경우 `특이사항`으로 통합
- `ISSUE 없음` → `없음`
- `근무현황` → `근무인원`
- 사업장 이름은 기존 HTML 표에 쓰는 기준 명칭 사용
- `수서 본사`는 생산 사업장이 아니므로 현재의 짧은 형식을 예외로 유지 가능
- `춘천두부/얼음`처럼 별도 근무자 표가 있는 경우, 그 표를 근무인원 산정의 기준으로 사용
- `음성물류`는 일반 공장과 달리 `오전/오후 근무인원`과 `작업진척률` 중심으로 정리
- `음성물류`에서 `근무인원`은 보고서에 `현장 근무인원` 표가 있으면 그 표의 `출근` 합계를 오전/오후 인원으로 사용
- `음성물류`에서 `작업진척률`은 오후 진척률 표 기준으로 정리하고, 보고서에 `계획물량`이 보이면 함께 포함

## 요약 방식

- 장문의 보고 문장은 짧은 항목형으로 압축
- 인사말, 서명, 반복 문장 제거
- 여러 줄이 필요하면 `-` 형식의 하위 줄 사용
- 하위 줄은 항상 하이픈 뒤에 한 칸 띄운 `- ` 형식으로 통일
- 최종 반영 전에는 하이픈 뒤 공백이 빠진 하위 줄이 없는지 한 번 더 확인
- 생산 세부 구분이 꼭 필요한 사업장만 `생산` 아래 하위 줄 사용
- 라인별 생산 보고라도 의미 손실 없이 줄일 수 있으면 `2. 생산 : 품목1, 품목2 ...` 형식으로 단순화
- 원문에 `없음`이 명시된 경우만 `없음` 사용
- 이슈가 있으면 `없음` 대신 실제 이슈를 적음
- 안전/환경 상태가 별도 이슈 없이 `없음`인 경우에는 독립 항목으로 남기지 말고 `특이사항 : 없음`으로 정리
- `생산` 항목 안에는 `특이사항 없음`을 남기지 않고 마지막 `특이사항` 항목으로 이동
- `생산` 항목에서는 품목명과 핵심 작업만 남기고, `생산중`처럼 없어도 의미가 유지되는 상태 표현은 제거
- 단순화된 `생산` 요약에서는 `라인운휴`, `휴무`, `생산없음` 같은 비생산 라인 정보는 꼭 중요하지 않으면 생략
- 의미 손실이 없으면 `전라인 가동 없음` 같은 표현은 `가동 없음`처럼 더 짧게 정규화
- 합계 수량과 세부 수량 합이 맞지 않으면 합계는 삭제하고, 세부 수량은 원문 그대로 유지
- `외부방문`은 방문 내용이 있으면 상세 내용을 적고, 없으면 반드시 `없음`으로 통일
- `특이사항`은 항상 마지막에 두고, 이슈가 없으면 `{번호}. 특이사항 : 없음` 형식으로 작성
- `춘천두부/얼음`은 `생산`을 `-두부공장`, `-얼음공장`으로 나눠 요약
- `춘천두부/얼음`은 대상 날짜의 주간/야간 표에서 이름이 채워진 근무자 항목 수를 세어 `근무인원`으로 사용
- `춘천두부/얼음` 보고에 다음날 외부업체 일정이 포함되면 `외부방문` 아래에 `두부공장(5/2 토)`처럼 사업장명 먼저, 날짜를 괄호로 붙여 정리

## 회색 처리 규칙

미업데이트 행은 아래처럼 `pending` 클래스를 사용합니다.

```html
<tr class="pending">
```

회색 스타일은 전체 행에 적용합니다.

```css
.pending td {
  color: #9a9a9a;
}
```

보고가 확인되어 반영되면 `pending` 클래스를 제거합니다.

## 수정 원칙

- 관련 없는 행은 건드리지 않음
- 이미 통일된 표현은 다시 바꾸지 않음
- 표 폭, 글꼴, 테두리는 요청이 없으면 유지
- 오후 업데이트 반영 시 실제로 달라진 항목만 수정
